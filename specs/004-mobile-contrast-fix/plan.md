# Implementation Plan: Mobile Contrast Fix

**Branch**: `004-mobile-contrast-fix` | **Date**: 2026-03-08 | **Spec**: `specs/004-mobile-contrast-fix/`

## Summary

Ensure all headings, labels, inputs, selects, textareas, and placeholders render with WCAG AA contrast on iOS Safari, regardless of the device's OS dark mode setting. The app does not support dark mode. Fix is achieved by (1) neutralizing Skeleton UI's dark mode CSS variable injection via `@custom-variant dark` redefinition (already in place), and (2) adding explicit Tailwind text-color classes to all form and heading elements that currently rely on CSS inheritance.

## Phase 0: Research — COMPLETE
See `research.md`. Key finding: `@custom-variant dark` successfully disables Skeleton's dark mode injection. Remaining contrast failures are caused by elements without explicit Tailwind text-color classes.

## Phase 1: Design — COMPLETE
See `data-model.md`. Color token map established. Four files need changes; each change is additive (no class removals).

## Implementation Approach

### Layer 1 — Global CSS (already done, `app.css`)
- `@custom-variant dark (&:where(.dark, .dark *))` — disables OS dark mode activation
- `color-scheme: light only` — signals light-only to browser
- Explicit color rules on `h1–h6`, `label`, `input`, `select`, `textarea` as catch-all

### Layer 2 — Explicit Tailwind classes (to implement)
Add per-element Tailwind color classes to HIGH and MEDIUM risk elements identified in the audit. This is more specific than the global rules and survives any future CSS changes.
